# Success Metrics — Group 3 (Christoph Dashboard)

These metrics define what "success" looks like for our Christoph Dashboard prototype during Week 4 usability testing. Metrics are split into quantitative (measurable during sessions) and qualitative (assessed via post-task questions and observation).

---

## 1. Usability & Cognitive Load (Quantitative)

| Metric | Target | How Measured |
|---|---|---|
| **Task Success Rate** | ≥ 80% tasks completed without assistance | Observer marks pass/fail per task per participant |
| **Time-on-Task (T1)** | Find key document ≤ 90 seconds | Stopwatch from task prompt to participant confirmation |
| **Time-on-Task (T2)** | Locate team contact ≤ 60 seconds | Stopwatch from task prompt to participant confirmation |
| **Click Efficiency** | Critical info reachable in ≤ 3 clicks | Observer counts clicks per task |
| **First-Click Accuracy** | ≥ 80% correct first clicks on navigation tasks | Observer marks correct/incorrect first click |
| **System Usability Scale (SUS)** | Score ≥ 80 | 10-question SUS survey completed post-session |
| **Error Rate** | ≤ 1 critical error per session | Observer notes navigation dead ends and wrong paths |

---

## 2. Feature Usefulness (Qualitative)

| Metric | Target | How Measured |
|---|---|---|
| **Confidence Score** | ≥ 4 / 5 | Post-task question: "How confident do you feel using this dashboard?" (1–5 scale) |
| **Trust Perception (AI)** | ≥ 4 / 5 | Post-task question: "Do you understand why the AI gave this answer?" (1–5 scale) |
| **Value Clarity** | User can state the platform's purpose in ≤ 60 seconds | Closing question: "In your own words, what does this platform do for you?" |
| **Perceived Reduction in Tool-Switching** | Positive verbal acknowledgement from ≥ 3/5 participants | Think-aloud observation during unified dashboard task |
| **Onboarding Relevance** | ≥ 3/5 participants say onboarding felt role-specific | Post-task question: "Did the onboarding feel relevant to a real job?" |

---

## 3. Trust & Transparency (AI-Specific)

| Metric | Target | How Measured |
|---|---|---|
| **Source Identification** | User can name source document in ≤ 30 seconds | Observer times identification from AI response screen |
| **AI Answer Acceptance Rate** | ≥ 3/5 participants say they would act on the AI answer | Post-task question: "Would you trust this answer enough to act on it?" |
| **Unprompted Trust Comments** | ≥ 2 positive unprompted comments about transparency | Think-aloud transcription review |

---

## 4. Baseline Comparison

Since HiveMind's MVP is pre-launch, we compare the prototype experience against the "current state" scenario described by participants in Week 1 interviews (e.g., "5–30 minutes searching multiple tools").

| Scenario | Current State (Interviews) | Prototype Target |
|---|---|---|
| Find compliance policy | 5–30 minutes, 3+ platforms | ≤ 2 minutes, 1 platform |
| Know who to ask for help | "Had to ask multiple people" | ≤ 2 clicks from dashboard |
| Understand AI response | "Don't know where it comes from" | Source visible, rated ≥ 4/5 trust |
| Know next onboarding task | "Not clear for 1–2 weeks" | Visible on dashboard, no prompt needed |

---

## 5. Data Collection Tools

- **Observer sheet:** per-task grid (task, success, time, clicks, notes)
- **SUS questionnaire:** printed or Google Form
- **Post-session questions:** 5 qualitative questions, rated 1–5
- **Think-aloud transcription:** audio recorded (with consent), reviewed for themes
